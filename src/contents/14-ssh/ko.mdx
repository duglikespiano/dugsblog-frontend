리눅스 머신, 개발환경등과 같은 클라우드 환경에서 일을할 때, **SSH (Secure Shell)** 를 접하게 됩니다.

SSH는 터미널을 통해 다른 네트워크나 컴퓨터에 안전하게 접속할 수 있도록 합니다.

서버 관리, 배포, 파일 전송 등에 사용되는 가장 보편적인 툴 중 하나입니다.

## Overview

| Feature            | SSH                    |
| ------------------ | ---------------------- |
| Full name          | Secure Shell           |
| Main purpose       | Remote computer access |
| Default port       | 22                     |
| Encryption         | ✅ Yes                 |
| Password login     | ✅ Supported           |
| Key authentication | ✅ Recommended         |

## What is SSH?

SSH is a network protocol that allows one computer to securely connect to another computer.

Without SSH, commands sent over a network could be exposed in plain text.

SSH encrypts all communication between client and server.

```txt
Your Computer ➡️ Internet ➡️ Remote Server
```

Because of encryption, usernames, passwords, and commands remain protected.

## Why Developers Use SSH

SSH is commonly used when managing cloud servers.

Typical examples include:

- connecting to a cloud VM
- deploying applications
- editing server files
- restarting services
- checking logs

For example, after creating a cloud server on providers like AWS, Amazon Web Services, Google Cloud, or DigitalOcean, SSH is usually the first way to access it.

## Basic SSH Command

The basic command is:

```bash
ssh username@server-ip
```

Example:

```bash
ssh ubuntu@192.168.1.10
```

Here:

- ssh starts the SSH client
- ubuntu is the remote username
- 192.168.1.10 is the server IP address

## Why SSH Keys Are Preferred

SSH supports password login, but key authentication is safer.

A key pair contains:

- private key (kept on your computer)
- public key (stored on the server)

When connecting, the server checks whether your private key matches the stored public key.

## Generate SSH Key

You can generate a key with:

```bash
ssh-keygen
```

This creates:

```bash
~/.ssh/id_rsa
~/.ssh/id_rsa.pub
```

- id_rsa = private key
- id_rsa.pub = public key

## Copy Public Key to Server

To allow login without password:

```bash
ssh-copy-id username@server-ip
```

This copies your public key into:

```bash
~/.ssh/authorized_keys
```

on the remote server.

## Common SSH Port

SSH uses port **22** by default.
Sometimes servers change the port for security.

Then you connect like:

```bash
ssh -p 2222 username@server-ip
```

## File Transfer with SSH

SSH also allows secure file transfer.

Using SCP:

```bash
scp file.txt username@server-ip:/home/ubuntu/
```

This uploads a file securely.

## Typical Developer Workflow

A common workflow looks like:

1. Create cloud server
2. Get public IP
3. Receive SSH key
4. Connect by SSH
5. Deploy application

SSH becomes the main entrance to your remote machine.

## Summary

SSH is a secure way to control another computer remotely.

It is essential when working with:

- cloud servers
- Linux machines
- deployments
- remote development

For developers, SSH is one of the most important networking tools to understand.
